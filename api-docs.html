Finance Dashboard API

A secure REST API for tracking personal and organisational financial records with JWT-based authentication and role-based access control.

Base URL

http://localhost:8080

Content Type

application/json

Authentication

Bearer JWT token in Authorization header

Token Expiry

24 hours from time of issue

Authentication

All endpoints under /records/** require a valid JWT token. Public endpoints are /auth/register and /auth/login.

After a successful login, copy the token value from the response and send it with every protected request as:

Authorization: Bearer <your-token>

Roles & Permissions

Every user is assigned one of three roles at registration (default: VIEWER). Roles control what data a user can see and modify.

Role Create Records View Records Dashboard Scope
ADMIN ✅ ✅ ✅ All users Full access across all users
ANALYST ✅ ✅ ✅ All users Read/write, sees all data
VIEWER ❌ ✅ ✅ Own only Read-only, own data only

Error Responses

All errors return a JSON body. Validation errors return a field-keyed map.

400
Bad Request
Validation failed or business rule violated
401
Unauthorized
Missing, expired, or invalid JWT token
403
Forbidden
Authenticated but role lacks permission
500
Server Error
Unexpected internal error
Example error bodies
// Single error (400 / 401 / 403 / 500) { "error": "Invalid credentials" } // Validation error (400) { "email": "Please provide a valid email address", "password": "Password must be between 8 and 100 characters" }

Auth Endpoints

Public endpoints – no token required.

🌐 Public – no token required
Request Body
Responses

POST /auth/register Content-Type: application/json { "email": "john@example.com", "password": "Secret@123", "fullName": "John Doe", "phoneNumber": "+911234567890" }

{ "userId": 1, "message": "User registered successfully", "token": null, "type": null }

{ "email": "Email already registered" }

🌐 Public – no token required
Request Body
Responses

POST /auth/login Content-Type: application/json { "email": "john@example.com", "password": "Secret@123" }
200 OK
Success Response
{ "userId": 1, "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxIiwiaWF0IjoxNzA...", "type": "Bearer", "message": "Login successful" }

Use the returned token as: Authorization: Bearer <token> in all subsequent requests.

Financial Record Endpoints

All endpoints require a valid Bearer token. Behaviour differs by role.

POST /records Create a new financial record ▶
🔒 JWT Required ADMIN & ANALYST only
Request Body
Responses
FieldTypeConstraintsDescription
amountnumberPositive decimal, requiredTransaction amount (always positive)
typestringINCOME or EXPENSETransaction direction
categorystringNon-blank, requiredFree-form label: Salary, Rent, Food…
transactionDatestringISO 8601 datetime, requiredBusiness date of the transaction
descriptionstringOptionalFree-text notes
Example Request
POST /records Authorization: Bearer <token> Content-Type: application/json { "amount": 85000.00, "type": "INCOME", "category": "Salary", "transactionDate": "2025-04-01T09:00:00", "description": "April salary" }
201 Created
Success Response
{ "id": 1, "amount": 85000.00, "type": "INCOME", "category": "Salary", "transactionDate": "2025-04-01T09:00:00", "description": "April salary", "createdAt": "2025-04-02T11:30:00", "userId": 1 }

GET /records List financial records ▶
🔒 JWT Required All roles

Returns all records for the authenticated user. ADMIN and ANALYST receive records for all users.

Example Request
GET /records Authorization: Bearer <token>
200 OK
Success Response
[ { "id": 1, "amount": 85000.00, "type": "INCOME", "category": "Salary", "transactionDate": "2025-04-01T09:00:00", "description": "April salary", "createdAt": "2025-04-02T11:30:00", "userId": 1 }, // ... more records ]
GET /records/type?type={INCOME|EXPENSE} Filter records by transaction type ▶
🔒 JWT Required All roles
Query ParamTypeRequiredValues
typestring✅ YesINCOME or EXPENSE
Example Requests
GET /records/type?type=INCOME Authorization: Bearer <token> GET /records/type?type=EXPENSE Authorization: Bearer <token>
200 OK
Returns filtered array (same shape as /records)
GET /records/date-range?start=&end= Filter records within a date range ▶
🔒 JWT Required All roles
Query ParamTypeRequiredFormat
startstring✅ YesISO 8601: 2025-03-01T00:00:00
endstring✅ YesISO 8601: 2025-03-31T23:59:59
start must not be after end — a 400 error is returned if the range is inverted.
Example Request
GET /records/date-range?start=2025-03-01T00:00:00&end=2025-03-31T23:59:59 Authorization: Bearer <token>
200 OK
Returns filtered array (same shape as /records)
400 Bad Request – inverted range
{ "error": "start date must not be after end date" }
GET /records/dashboard Get income, expense and net balance summary ▶
🔒 JWT Required All roles

Returns aggregated financial totals and a category breakdown. ADMIN / ANALYST see totals for all users. VIEWER sees their own totals only.

Example Request
GET /records/dashboard Authorization: Bearer <token>
200 OK
Success Response
{ "totalIncome": 90000.00, "totalExpense": 24400.00, "netBalance": 65600.00, "categorySummary": { "Salary": 85000.00, "Freelance": 5000.00, "Rent": 12000.00, "Food": 3500.00, "Utilities": 1200.00, "Transport": 2200.00, "Shopping": 4500.00, "Entertainment": 1500.00 } }
FieldTypeDescription
totalIncomenumberSum of all INCOME records in scope
totalExpensenumberSum of all EXPENSE records in scope
netBalancenumbertotalIncome − totalExpense
categorySummaryobjectMap of category → total amount (income + expense combined per category)
